Elementos da Rede

  • Switch: Faz a conexão entre os dispositivos dentro da rede local (LAN).
  • Roteador: Conecta a rede local à internet ou outra rede externa.
  • Servidor: Responsável por fornecer serviços como DNS, HTTP e DHCP.
  • Clientes: Computadores ou dispositivos que consomem os serviços fornecidos pelo servidor.
  • Cabos de Rede: Conectam os dispositivos fisicamente.


    • Configuração no Cisco Packet Tracer:

  • Conecte o roteador ao switch com cabo Straight-Through.
  • Conecte o servidor e os PCs ao switch.
  • Configure o servidor com IP estático.
  • Configure os serviços DNS, HTTP e DHCP no servidor.
  • Configure os PCs para obter IP automaticamente (DHCP) ou configure manualmente.

Configuração dos Servidores

Servidor de Domínio (DNS):

Função: Traduz nomes de domínio (ex.: www.exemplo.com) em endereços IP.

Passos no Packet Tracer:

  • Abra o servidor.
  • Habilite o serviço DNS.
  • Adicione registros DNS:
  • Nome do domínio: www.exemplo.com
  • Endereço IP: 192.168.1.100

    • Servidor de Comunicação Web (HTTP):

    Função: Oferece páginas e serviços web acessíveis por navegadores.

    Passos no Packet Tracer:

  • No servidor, habilite o serviço HTTP e/ou HTTPS.
  • Configure a página inicial no editor interno do Packet Tracer.

    • Servidor DHCP:

    Função: Atribui automaticamente IP, máscara de sub-rede, gateway e DNS aos dispositivos na rede.

    Passos no Packet Tracer:

  • Habilite o serviço DHCP no servidor.
  • Configure o escopo de endereçamento:
  • Faixa de IP: 192.168.1.10 - 192.168.1.50
  • Máscara de sub-rede: 255.255.255.0
  • Gateway padrão: 192.168.1.1
  • DNS: 192.168.1.100

Equipamentos Aplicados em Redes: Funções e Configuração

Roteador:

  • Função: Direciona pacotes entre redes distintas.
  • Configuração: Atribua IPs às interfaces, configure NAT e roteamento estático/dinâmico.

    • Switch:

  • Função: Conecta dispositivos da mesma LAN.
  • Configuração: Crie VLANs para segmentação de rede.

    • Servidor:

  • Função: Fornece serviços como DNS, DHCP e HTTP.
  • Configuração: Ajuste os serviços desejados conforme necessário.

    • Access Point:

  • Função: Fornece conectividade Wi-Fi aos dispositivos.
  • Configuração: Configure o SSID e segurança (ex.: WPA2).

    • Firewall:

  • Função: Protege a rede ao filtrar pacotes e bloquear ameaças.
  • Configuração: Defina regras de permissão e bloqueio para IPs e portas.

    • Conceitos de Servidor e Protocolos

    Servidor:

  • Computador ou dispositivo que oferece serviços a outros dispositivos (clientes) em uma rede.

    • DNS (Domain Name System):

  • Converte nomes de domínio legíveis (ex.: google.com) em endereços IP (ex.: 142.250.64.110).

    • HTTP (Hypertext Transfer Protocol):

  • Protocolo para transferência de dados web. Utiliza portas 80 (HTTP) e 443 (HTTPS).

    • DHCP (Dynamic Host Configuration Protocol):

  • Protocolo que distribui automaticamente endereços IP, máscara de sub-rede, gateway e DNS para os dispositivos conectados à rede.

    • Modelo OSI e TCP/IP


    Modelo OSI (Open Systems Interconnection):

    Divide a comunicação em 7 camadas:
  • Física: Transmissão de bits.
  • Enlace: Comunicação entre dispositivos locais.
  • Rede: Roteamento de pacotes (ex.: IP).
  • Transporte: Controle de fluxo e erros (ex.: TCP, UDP).
  • Sessão: Gerencia conexões.
  • Apresentação: Formata os dados (ex.: criptografia).
  • Aplicação: Interface com o usuário (ex.: HTTP, FTP).

    • Modelo TCP/IP:


    Modelo de comunicação usado na internet, com 4 camadas:

  • Rede (Acesso): Combina as camadas física e de enlace do OSI.
  • Internet: Equivale à camada de rede (IP, ICMP).
  • Transporte: Gerencia dados entre dispositivos (TCP, UDP).
  • Aplicação: Oferece serviços diretamente ao usuário (HTTP, DNS, FTP).
